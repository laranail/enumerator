# v1.0.0 roadmap — laranail/enumerator

Forward-looking planning doc, drafted 2026-05-16. Anchors the v0.x
trajectory by making three things visible:

1. The **public surface that's stable enough to lock** at v1.0.0 (no
   pending design questions, real consumer usage, comprehensive tests
   pinning the contract).
2. The surface that's still **experimental** — needs more bake time
   in v0.x.
3. The **breaking changes worth saving for v1.0.0** so they don't
   accumulate as ad-hoc tech debt across patch releases. ADR-0001
   committed us to BC-hard within v1, so anything in this bucket
   sits behind the v1.0 gate.

This is a draft. A focused audit session should run before locking the
final v1.0.0 scope.

**Anchoring constraints:**

- ADR-0001: BC hard within v1. No shipped public-API removal or rename
  in v0.x. Anything that needs renaming, restructuring, or
  signature-tightening lives here.
- All 75 baked decisions in NEXT-SESSION's `D34..D75` set must hold
  through v1.0. They define the package's identity.
- PHP `^8.3` / Laravel `^13.0` floor stays through v0.x. v1.0.0
  can re-evaluate (PHP 8.4-only? Laravel 14-only?).

## What's still experimental (needs v0.x bake time)

These have shipped but need at least one more cycle of consumer
feedback or test-matrix coverage before locking.

| Surface | Why still experimental | What needs to happen |
|---|---|---|
| `Integrations\Livewire\WithEnumTransitions` | Shipped v0.3.0, extended v0.4.0 (PR-ν). Real-Livewire-roundtrip tests still missing. | Add Livewire's request-lifecycle harness, exercise bulk + validation helpers against a real component. |
| `Integrations\Livewire\EnumeratorCasts` | hydrateProperty() signature is reasonable but only one consumer pattern verified. | Verify against pure-enum + class-const + multi-locale + nested-property paths. |
| `Modules\StructuredOutput\*SchemaEmitter` (Anthropic/Mcp/OpenAi) | Three vendor-specific emitters; specs evolve outside our control. | One more cycle of vendor-spec verification; consider adding a `GenericJsonSchemaEmitter` to absorb future vendors. |
| `Modules\Lighthouse\EnumScalar` | Only path-scoped tested; PHPStan excludes the dir. | Bring under PHPStan analysis + add Testbench-Lighthouse fixture. |
| `Modules\Saloon\EnumCaster` | Limited consumer coverage. | Real-Saloon integration test. |
| `Modules\Octane\WarmCachesListener` | Coverage 62.5 % even after PR-ι. The bind path runs once at boot; harder to test. | Octane-Testbench harness, or accept the gap and document. |
| `Modules\Pest\Expectations` | Single set shipped; consumer usage minimal. | Verify the higher-order expectation chain with a downstream Pest plugin. |
| `Integrations\Filament` / `Integrations\Inertia` / `Integrations\Nova` | PHPStan-excluded — these integrations are scaffolds, not real code yet. | Decide: ship for real (PR sequence in a future cycle) or drop. |

Each experimental surface follows the same discipline: it stays
**under** `Integrations\*` or `Modules\*` (not in the core), and
documentation labels it as experimental until v1.0.

## Breaking changes worth saving for v1.0.0

Things that would make the package cleaner if we could break them,
but can't until v1.0. Pile them up here so v1.0.0's CHANGELOG has
a clean migration list.

### Method renames / signature tightening

| Site | Today | v1.0 target | Why |
|---|---|---|---|
| `LaravelTranslatorAdapter::has()` | `function_exists('trans')` guard | drop the guard | Lang is always bound in Laravel ≥ 7; the guard is dead defensive code that only matters outside any Laravel context. |
| `LaravelTranslatorAdapter::getLocale()` | `'en'` literal fallback | drop the fallback | Same reason — `app()` is always callable inside Laravel. |
| `Support\BladeViewHelpers::toScalar()` | `is_scalar($value) ? $value : (string) $value` | tighter union return | The `mixed` parameter widens harder than needed; could narrow to `BackedEnum\|UnitEnum\|scalar\|object`. |
| `Concerns\HasEnumeratorBehavior::has(target)` | static, accepts `mixed` | could narrow to `string\|int\|BackedEnum\|UnitEnum` | Wider type opens unnecessary failure surface. |
| `Concerns\HasTransitions` parameter types | `self $target` on `canTransitionTo` / `transitionTo` / `tryTransitionTo` | widen to `Stateful $target` | v0.5.0 PR-β2 added the six instance methods to `Contracts\Stateful` as `@method` PHPDoc rather than PHP-level signatures because the trait's `self` parameter is narrower than `Stateful` — PHP's contravariance rule rejects the override (`Declaration of OrderStatus::canTransitionTo(OrderStatus) must be compatible with Stateful::canTransitionTo(Stateful)`). Widening the trait's parameter type to `Stateful` lets the interface carry full PHP-level signatures, retiring the `@method` PHPDoc workaround. Existing call sites (`$enum->canTransitionTo($otherCase)`) keep working — `self` is a subtype of `Stateful`. The break is in test fixtures that override the method with a narrower signature. |

### Trait composition changes

| Site | Today | v1.0 target | Why |
|---|---|---|---|
| `HasEnumerator` umbrella | composes 7 traits as v0.3.0 | optional sub-umbrella `HasMinimalEnumerator` (`Has*` only, no `Is*`) | Some consumers want just attributes + helpers without the rendering / json / translation layer. Splitting the umbrella in v1 lets that work without dropping a trait. |
| `Concerns\HasTransitions` | always implicitly composed via `HasEnumerator` | break apart so it's opt-in only | Today `HasEnumeratorBehavior` brings in `HasTransitions`, which means every enum has `transitions()` / `initialStates()` referenced in PHPStan even when the enum doesn't implement `Stateful`. v1 makes this opt-in. (See D65 path-scoped suppression — that becomes unnecessary if the trait is opt-in.) |

### Config-key cleanup

| Today | v1.0 target | Why |
|---|---|---|
| `enumerator.magic.allow_invokable_cases` default `false` | switch default to `true` | The invokable-cases path has shipped for 4 cycles, all known callers want it on. Default flip is a behaviour break for the (probably zero) consumers who relied on `false`. |
| `enumerator.cache.driver = 'layered'` | rename to `enumerator.cache.strategy` | "Driver" overloads Laravel's cache driver naming and confuses readers. "Strategy" is unambiguous. |
| `enumerator.alpine.cdn_url` template placeholder `{version}` | switch to standard `:version` (Laravel-style) | Consistency with Laravel's route placeholder syntax. |

### Removed-but-shipped APIs

None at present. v0.x has so far been strictly additive (per
ADR-0001). Audit before v1 to make sure nothing slipped.

## v0.x → v1.0 sequence

Cycle | Theme | Required to hit v1.0?
---|---|---
v0.3.0 | Integration depth (Livewire + cache + multi-select) | shipped pre-tag
v0.4.0 | Coverage + parity + hardening | in progress (see v0.4.0-scope.md)
v0.5.0 | Experimental → first-class: at least one of Lighthouse / Saloon / a full Filament integration | required
v0.6.0 | i18n contributor scaffolding + community translations (re-evaluate ADR-0003) | optional but valuable
v0.7.0 | Browser-level Alpine multi-select a11y verification + Dusk smoke test | required
v0.8.0 | v1 prep — sweep `experimental` labels, classify final breaking-change set, write migration doc | required
v0.9.0-rc | v1.0.0 candidate: signature freeze, CHANGELOG migration block, deprecation notices for v0.x callers | required
v1.0.0 | Apply breaking changes from this doc; flip the BC gate; publish migration guide | terminal

Each cycle's actual scope is decided per-session; the above is the
**latest-not-earlier** ordering — items can shift earlier but
shouldn't shift later without re-cutting this doc.

## Manual checklist before tagging v1.0.0

1. `docs/migration/v0.x-to-v1.0.md` — full migration guide.
2. `docs/README.md` "since version" badges on every tool page.
3. Architecture tests (`tests/Architecture/`) pinning every locked
   contract / trait / cast signature via reflection. Add `Stability`
   tests modelled on Laravel's own.
4. Re-run the cross-project PHP-level smoke test on a fresh Laravel
   skeleton (D72). v1.0 must boot from `composer require` without
   surprises.
5. Update Simtabi OSS portal pages (`opensource.simtabi.com/products/
   laranail/enumerator` + `documentation/laranail/enumerator`) for
   the v1.0 milestone.
6. Tag `v1.0.0` on a clean `main`, manually edit the GitHub release
   notes with the migration block.
